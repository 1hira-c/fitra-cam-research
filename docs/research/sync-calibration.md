# 同期とキャリブレーション設計

## 結論

- 3D 精度に最も効くのは **2D モデルの差よりも、フレーム同期と外部キャリブレーション品質**。
- USB UVC カメラでは厳密なハード同期が難しいため、初期構成は **ソフト同期 + タイムスタンプ + 可視同期イベント** を前提にする。
- キャリブレーションターゲットは **ChArUco を第一候補**にするのが安全。

## 同期方針

### 現実的な初期案

1. 各カメラのキャプチャ直後に **monotonic timestamp** を付与する。
2. キャプチャバッファを浅くし、**最新フレーム優先**で処理する。
3. 計測開始時に **LED フラッシュなど全視点に見える同期イベント** を入れる。
4. 長時間計測では、必要に応じて途中でも同期イベントを入れて **ドリフト補正**できるようにする。

### 同期精度の目安

- 30 fps では 1 フレームが約 33 ms。
- 実用上は **サブフレーム誤差**を狙い、目安として **5–10 ms 以下**に収めたい。
- これは厳密な保証値ではなく、動きが速いほど要求は厳しくなる。

## キャリブレーション方針

### 推奨

- **内部パラメータ**: 各カメラ単体で取得
- **外部パラメータ**: 固定設置後に全カメラを同一ワールド座標へ合わせる
- **ターゲット**: ChArUco 優先、全視点で見やすければ checkerboard でも可

### ChArUco を推す理由

- 部分的にしか見えなくても検出しやすい
- マーカー ID により対応点の曖昧さが減る
- 多視点・広角配置で運用しやすい

## MMPose / 3D 化への影響

- 多視点三角測量では、各カメラの **intrinsic / extrinsic** が必須。
- パラメータは一貫したワールド座標系で保存する必要がある。
- MMPose はカメラ校正自体は提供しないため、OpenCV などで外部作成した結果を渡す前提で考える。

## 実装メモ

1. 初期段階では **2 台**で内部・外部校正フローを固める。
2. 校正保存形式は `camera_id -> intrinsics/extrinsics` の JSON/YAML を想定する。
3. 推論時は各フレームに timestamp を添え、**最も近い時刻の組を束ねる**。
4. 三角測量前に、各視点の検出信頼度しきい値と欠損補間方針を決める。

## 最初の成立条件

- 2 台のカメラで同期イベントを使った時刻合わせができる。
- ChArUco か checkerboard で内部・外部パラメータを取得できる。
- 同一人物の同時刻 2D キーポイントを対応付けできる。
